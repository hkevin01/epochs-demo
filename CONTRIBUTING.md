# Contributing

## Setup

1. Create and activate a virtual environment.
2. Install dependencies with `pip install -r requirements.txt`.
3. Run the project entry point you are changing and verify the output locally.

## Pull requests

1. Keep changes focused and small.
2. Update documentation when behavior, commands, or repository structure changes.
3. Include the commands you used for validation in the pull request description.

## Issues

Use the issue templates to report bugs and feature requests.
